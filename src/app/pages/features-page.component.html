One Schema For All

With Deepkit Type you can define your data structures once and use them in your whole application stack. Use the same schema API for all use-cases where serialization/validation is required.

For example you could define an entity User once, decorate it with all necessary information like data type, relationship information, and validation. Once defined, you can then use the very same class for database, frontend, http route (body) including validation, and more.

Class slicing enables you to include/exclude fields allowing you to create new sub classes in DRY-way — never write entities more than once.

Type-safe Configuration System

The most type-safe configuration system with automatic deserialization and validation. Consume configuration values directly in your services and controllers in the most type-safe and decoupled way possible.

Configuration values can be set via env variables, dotEnv files, or manually in code.
